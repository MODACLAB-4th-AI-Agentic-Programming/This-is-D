# 장한준 - 2주차

**이번주 주제** : 에이전트를 나누고 역할 부여 실습

## 사전 지식

### 별개의 에이전트로 개발
- 기획 - 개발 의 각 폴더에서 Agent 가 나뉘어 개발함
	- 실제 회사에서의 기획자 - 개발자 간 소통과 유사하게 블랙박스식 소통을 의도
	- 해당 방안으로 했을 시 구현시에는 큰 문제가 없지만 기획 에이전트가 개발의 내용을 몰라 발생하는 일이 많았음

---

## 실습 과정

바로 사이드 프로젝트에 투입하기에는 미숙한 숙련도로 인해 코드가 더러워질까봐 별개의 작업을 진행했습니다.
게임 기획용 멀티 에이전트 프리셋이 있다 하여 옵시디언에서 시도해 보았습니다.

### 구조 

[하네스 - 100](https://github.com/revfactory/harness-100/tree/main/ko)

```
.claude/
├── agents/
│   ├── worldbuilder.md        — 세계관 설계자 (배경, 세력, 역사, 규칙)
│   ├── quest-designer.md      — 퀘스트 디자이너 (메인/사이드 퀘스트, 보상 체계)
│   ├── dialogue-writer.md     — 대사 작가 (NPC 대사, 선택지, 감정 연출)
│   ├── branch-architect.md    — 분기 설계자 (분기 구조, 결말, 플래그 시스템)
│   └── narrative-reviewer.md  — 내러티브 검증자 (정합성, 밸런스, 품질 검증)
├── skills/
│   ├── game-narrative/
│   │   └── skill.md           — 오케스트레이터 (팀 조율, 워크플로우, 에러핸들링)
│   ├── quest-design-patterns/
│   │   └── skill.md           — quest-designer 확장 (12 아키타입, DRIP 보상, 난이도 곡선)
│   ├── dialogue-systems/
│   │   └── skill.md           — dialogue-writer 확장 (VOICE 보이스, 선택지 심리학, 바크)
│   └── branching-logic/
│       └── skill.md           — branch-architect 확장 (6가지 분기 패턴, 플래그, 엔딩)
└── CLAUDE.md                  — 이 파일
```

## 특이한 점 / 좋았던 점 / 나빴던 점

1. 해당 멀티 agent 들 조차 AI 에게도 만들어줄 수 있기에 좋았습니다
2. 게임 개발의 경우 다 클래 개발자라 칠 경우 개발 자체로는 멀티 에이전트를 쓰면 오히난잡해질 느낌이 강합니다
	- 때문에 QA 쪽에서는 멀티 에이전트를 활용해 어떤 부분들을 QA 할지 뽑아볼 수 있을 것 같아 기대됩니다.
	- 코드리뷰 또한 가능할 것 같은데 이 경우 다른 AI (Claude-Gemini 등) 끼리의 소통이 좋다 해 시도해볼 예정입니다.
