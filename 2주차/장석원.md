# Unity MCP 활용 사례 발표: AI가 바꾼 Unity 게임 개발 워크플로우

Unity 에디터와 AI를 연결하는 **Unity MCP**를 통해 단순 반복 작업을 자동화하고, 개발 효율을 극대화한 사례를 정리한 문서입니다.

---

## 1. Unity MCP — 3가지 핵심 기능
Unity MCP는 Unity 에디터와 AI(Claude 등)를 연결해주는 서버로, 다음과 같은 기능을 수행합니다.

* **씬 조작 자동화**: AI가 Hierarchy에 직접 접근하여 게임 오브젝트를 생성하고 배치합니다.
* **컴포넌트 자동 연결**: 스크립트 작성, 컴포넌트 추가(Add), Inspector 초기값 세팅까지 자동으로 처리합니다.
* **에셋 검색 및 관리**: 프리팹·머티리얼의 위치와 상태를 AI가 직접 읽고 수정 방향을 제시합니다.
* **결과**: 개발자는 반복적인 클릭 작업에서 해방되어 핵심 로직 설계와 최적화에만 집중할 수 있습니다.

---

## 2. 도입 효과: 수동 방식 vs Unity MCP
| 구분 | Before (수동 방식) | After (Unity MCP) |
| :--- | :--- | :--- |
| **작업 흐름** | 코드 복사 및 에디터 붙여넣기 | AI가 에디터 환경 직접 조작 |
| **에디터 설정** | 컴포넌트 연결 및 Inspector 값 수동 세팅 | 씬 자동 생성 및 컴포넌트 속성 자동 설정 |
| **에셋 관리** | 반복 클릭 및 확인 작업 | 에셋 검색 및 상태 직접 파악 |
| **핵심 가치** | 단순 작업에 시간 소모 | **핵심 로직에만 집중 가능** |

---

## 3. 토큰 최적화 솔루션 (문제 해결 방안)
방대한 에디터 컨텍스트로 인한 **토큰 비용 폭등** 문제를 해결하기 위한 세 가지 전략입니다.

### ① 프리팹 단위 작업
* 씬 전체를 조회할 경우 오브젝트 수에 따라 JSON 데이터가 거대해집니다.
* **해결**: 필요한 오브젝트와 그 자식만 포함된 **프리팹 단위로 작업**하여 데이터 전송량을 최소화하고 토큰 소모를 절감합니다.

### ② 작업 단위로 대화 끊기
* 하나의 대화에서 여러 작업을 이어서 진행하면 이전 결과가 컨텍스트에 쌓여 토큰 소모가 누적됩니다.
* **해결**: 작업 단위로 대화를 새로 시작하여 **컨텍스트를 초기화**함으로써 입력 토큰을 최소화합니다.

### ③ Serena MCP 활용 (LSP 기반 심볼 분석)
* 전체 파일을 통째로 읽는 방식은 불필요한 컨텍스트 전송으로 토큰을 낭비합니다.
* **해결**: **Serena MCP**를 도입하여 Class/Method 단위의 정밀 심볼 분석을 수행합니다.
* `find_symbol`로 정확한 위치만 선택하여 필요한 코드만 로딩합니다.
