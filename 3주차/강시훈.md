## 1. AI 활용 전략

### 1-2. AI 도구별 상세 활용 계획

**Codex (코드 제작 및 작업 설계 전담)**

Codex는 이번 프로젝트에서 코드 제작자이자 작업 설계자 역할을 맡는다. 기존 문서에 정리된 핵심 루프를 기준으로, 2주 안에 실제로 플레이 가능한 프로토타입 범위를 정하고 Unreal C++ 구현 작업을 보조한다.

- Unreal C++ 클래스 작성 및 수정
- 리슨 서버 세션 흐름 구현 보조
- `GameMode`, `GameState`, `PlayerState` 역할 분리 정리
- 인벤토리, 자원, 레버, 정산 시스템 구현 방향 설계
- 멀티플레이 복제 오류, ServerTravel 문제, 상호작용 오류 분석
- 제출용 기획 문서 및 일정표 작성
- 기능별 테스트 체크리스트 작성

**Claude 연동 레벨디자인 에이전트 (맵 설계 전담)**

레벨디자인은 별도로 제작한 전용 에이전트를 활용한다. 이 에이전트는 코드 구현보다 행성 맵의 구조, 동선, 배치, 위험 리듬을 설계하는 데 집중한다.

- 폐행성 외부 지역 1개 설계
- 내부 시설 1개 설계
- 함선 착륙 지점과 복귀 동선 설계
- 배달 수령 지점 배치
- 프라이팬, 톱니바퀴, 엔진 자원 배치
- 내부 추적형 적, 외부 귀환 방해형 적, 환경 위협 배치
- 초반 탐색 → 중반 욕심 → 후반 철수 압박 흐름 설계

### 1-3. AI 제작 파이프라인 구조

#### 초기 접근: Codex 단독 기획 및 구현 보조

초기에는 Codex를 중심으로 기존 기획 문서를 읽고, 핵심 기능을 정리하고, 개발 우선순위를 산정하는 방식으로 작업을 진행한다. 이 방식은 코드와 문서를 빠르게 정리하기에는 적합하지만, 레벨디자인까지 모두 Codex가 담당하면 작업 범위가 넓어지고 맵 설계의 집중도가 떨어질 수 있다.

따라서 프로토타입 제작에서는 역할을 분리한다.

- **Codex:** 코드, 시스템 설계, 문서, 일정 관리
- **Claude 연동 레벨디자인 에이전트:** 맵 구조, 동선, 자원/위협 배치
- **사용자:** 최종 결정, 에디터 확인, 플레이테스트

#### 전환: 코드 설계와 레벨디자인 병렬 진행

2주 프로토타입에서는 시스템 구현과 레벨디자인을 병렬로 진행한다. Codex가 현재 구현 가능한 기능 목록을 정리하면, 레벨디자인 에이전트가 그 기능이 실제로 놓일 행성 맵 구조를 설계한다. 이후 사용자가 방향을 결정하고, Codex가 구현 작업 단위로 다시 분해한다.

**에이전트 역할 구성:**

| 에이전트 | 구현 방식 | 역할 |
|---------|----------|------|
| Codex | 메인 작업 세션 | 코드 제작, 문서 정리, 작업 분해, 일정 산출, 버그 분석 |
| AI 레벨디자이너 | Claude 연동 전용 에이전트 | 행성 맵 구조, 동선, 자원 배치, 적/환경 위협 배치 |
| 사용자 | 직접 검수 | 기획 판단, 언리얼 에디터 확인, 플레이테스트, 최종 승인 |

**파이프라인 흐름:**

```text
[Codex - 메인 작업 설계]
핵심 기능 분석, 구현 우선순위 산정, 현재 구현 가능 기능 목록 정리
    │
    ├── 코드 구현 방향 수립
    │       ↓
    │   Unreal C++ / Blueprint 연결 / 멀티플레이 복제 검증
    │
    └── 레벨디자인 에이전트에 맵 설계 요청
            ↓
        행성 맵 블록아웃 / 자원 배치 / 위협 배치 / 철수 동선 설계
            ↓
        사용자 검토 및 방향 결정
            ↓
        Codex가 구현 작업 단위로 재분해
            ↓
        통합 테스트 및 3일 루프 검증
```

#### 단일 AI 작업 vs 역할 분리 파이프라인 비교

| 항목 | 단일 AI 작업 | 역할 분리 파이프라인 |
|------|-------------|--------------------|
| 작업 범위 | 코드, 기획, 레벨을 한 세션에서 모두 처리 | 코드/기획은 Codex, 레벨은 전용 에이전트가 담당 |
| 집중도 | 작업 맥락이 넓어질수록 품질 저하 가능 | 각 AI가 자기 역할에 집중 |
| 진행 방식 | 순차 진행 중심 | 코드 설계와 레벨디자인 병렬 진행 가능 |
| 결과물 | 문서와 코드 중심 | 문서, 코드, 맵 설계안이 동시에 축적 |
| 2주 완성 가능성 | 기능 범위가 흔들릴 위험 있음 | 핵심 루프 중심으로 범위 통제 가능 |

**이번 작업 목표:**

- Codex로 코드 제작과 기획 문서 제작 진행
- Claude 연동 레벨디자인 에이전트로 행성 맵 1종 설계
- 2주 안에 핵심 루프가 끝까지 도는 프로토타입 완성
- Steam/EOS, 다중 행성, 고급 적 AI는 프로토타입 이후로 제외

## 4. 프로젝트 작업 계획 요약

### 핵심 목표

> `Operation X`를 문서 기반 기획 단계에서 **멀티플레이로 핵심 루프가 작동하는 프로토타입**으로 전환한다.  
> 플레이어가 행성으로 내려가 자원을 회수하고, 함선 레버로 하루를 끝내며, 3일 뒤 정거장에서 목표 크레딧을 정산하는 흐름을 완성한다.

### 프로토타입 포함 범위

- 리슨 서버 세션 생성 및 참가
- 함선 허브 입장과 행성 출발
- 행성 맵 1종 진입
- 자원 3종 줍기, 버리기, 함선 반입
- 인벤토리 6칸과 무게 기반 이동 저하
- 스태미나 소모 및 회복
- 함선 레버로 하루 종료
- 제한 시간 종료 시 자동 출발
- 3일 루프 진행
- 정거장 정산과 목표 크레딧 판정
- 기본 권총, 탄창, 빠루, 프라이팬 공격
- 구매 및 배달 최소 흐름
- 적대 위협 2종과 환경 위협 1종의 최소 버전
- 핵심 상태를 보여주는 UI

### 프로토타입 제외 범위

- Steam 또는 EOS 정식 연동
- 호스트 마이그레이션
- 전용 서버
- 다중 행성
- 복잡한 적 AI와 보스형 위협
- MP5, 수류탄, 섬광탄, 스턴건
- 정거장 생활형 콘텐츠
- 고급 상점 UI
- 완성형 아트, 사운드, 애니메이션 폴리싱

## 5. AI 활용 핵심 포인트 정리

이번 프로젝트에서 AI는 단순한 보조 도구가 아니라 **2주 프로토타입 제작 파이프라인의 핵심 구성요소**로 기능한다.

1. **Codex 중심 코드 제작**: Codex는 Unreal C++ 구현, 기존 코드 수정, 빌드 오류 분석, 멀티플레이 복제 구조 정리를 맡는다.
2. **Codex 중심 문서 제작**: 기존 Markdown 기획 문서를 읽고, 제출용 문서와 일정표, 기능별 시간 산출표를 작성한다.
3. **레벨디자인 역할 분리**: Claude 연동 전용 에이전트가 행성 맵 1종의 구조, 동선, 자원 배치, 위협 배치를 전담한다.
4. **병렬 작업 구조**: Codex가 시스템 구현을 진행하는 동안 레벨디자인 에이전트가 맵 설계를 병렬로 수행하여 2주 일정을 압축한다.
5. **프로토타입 범위 통제**: AI를 활용하더라도 핵심 루프와 무관한 Steam/EOS, 다중 행성, 고급 적 AI, 완성형 아트/사운드는 제외한다.
6. **서버 권한 우선 원칙**: 레버, 자원 반입, 정산, 사망, 손실 규칙은 서버가 최종 판정하도록 설계한다.
7. **검증 중심 개발**: 목표는 많은 콘텐츠가 아니라, 멀티플레이 상태에서 3일 루프가 끝까지 도는지 확인하는 것이다.

## 최종 정리

`Operation X`의 2주 목표는 완성형 게임 제작이 아니라, 협동 추출 게임의 핵심 판단이 실제 플레이에서 발생하는지 검증하는 것이다.

Codex는 코드 제작과 기획 문서 제작을 담당하고, Claude 연동 레벨디자인 에이전트는 행성 맵과 위험 배치를 담당한다. 사용자는 최종 방향을 결정하고 플레이테스트를 통해 밸런스를 확인한다.

최종 산출물은 다음과 같다.

- 실행 가능한 Operation X 프로토타입
- 호스트/클라이언트 기준 플레이테스트 결과
- 핵심 기능 구현 목록
- 제외한 기능 목록과 제외 사유
- AI 활용 내역
- 2주 작업 시간 산출표
- 다음 개발 단계 제안
