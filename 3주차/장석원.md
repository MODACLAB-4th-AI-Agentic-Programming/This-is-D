# 3주차: 심화주제 탐색 & 프로젝트 설계

## 프로젝트 개요

**프로젝트명:** AI 기반 인디 액션 게임 제작  
**현재 상태:** 프리미티브 기반 POC(Proof of Concept) 완성  
**핵심 피처:** 근접전 / 불렛 패링 / 불렛타임  
**개발 환경:** Unity + UnityMCP (Claude 연동)

## 1. AI 활용 전략

### 1-1. 활용할 AI와 역할 분담

| AI 도구 | 활용 분야 | 활용 방식 |
|---------|----------|----------|
| Claude (UnityMCP) | 레벨 디자인 | Unity 씬에 직접 프리팹 배치 및 맵 구성 |
| Claude | 기획 · 오케스트레이션 | 멀티 에이전트 구조로 기획→레벨디자인→아트 프롬프트 파이프라인 운영 |
| 나노바나나2 (NanoBanana2) | 아트 에셋 | 캐릭터, 배경, UI, 이펙트 등 2D 에셋 생성 |
| Suno | 사운드 | BGM 및 SFX 제작 |

### 1-2. AI 도구별 상세 활용 계획

**나노바나나2 (아트 에셋 전담)**

나노바나나2를 선택한 이유는 게임 에셋에 적합한 스타일 제어가 가능하고, 일관된 아트 톤을 유지하면서 빠른 반복 생성이 가능하기 때문이다. 오케스트레이션에서 AI 아트 담당이 기획 요구사항을 바탕으로 아트 프롬프트를 생성하고, 나노바나나2에 입력하여 에셋을 일괄 생성한다.

- 캐릭터 스프라이트 (플레이어, 적 유형별)
- 배경 및 환경 타일셋
- UI 요소 (HUD, 메뉴, 아이콘)
- 이펙트 스프라이트 (불렛, 패링 이펙트, 불렛타임 연출)

**Suno (사운드 전담)**

Suno를 활용하여 게임 분위기에 맞는 BGM과 효과음을 생성한다. 텍스트 프롬프트로 장르·분위기·BPM을 지정하여 게임 상황별 사운드를 빠르게 제작할 수 있다.

- BGM: 메인 타이틀, 전투, 보스전, 메뉴 화면
- SFX: 근접 타격음, 불렛 패링 사운드, 불렛타임 진입/해제 효과음
- 앰비언스: 스테이지별 배경 환경음

### 1-3. 멀티 에이전트 오케스트레이션 구조 (2주차 연계)

#### 초기 접근: 단일 세션 프롬프트 기반 역할 전환

2주차에서는 Claude.ai 단일 세션 안에서 프롬프트로 역할을 전환하는 방식으로 오케스트레이션을 시도했다. 기획자 모드에서 요구사항을 정리하고, 레벨디자이너 모드로 전환하여 UnityMCP로 맵을 설계하고, 아트 담당 모드에서 프롬프트를 생성하는 식이었다.

그러나 실제 작업을 진행하면서 다음과 같은 한계가 드러났다.

- **컨텍스트 오염**: 기획·아트·레벨·개발 작업이 하나의 세션에 섞이면서 대화가 길어질수록 앞선 맥락을 잃고 프롬프트 품질이 저하됨
- **병렬 처리 불가**: 아트 에셋 생성과 레벨 배치를 동시에 진행할 수 없어 순차적으로만 작업해야 했고, 대기 시간이 누적됨
- **역할 경계 모호**: 단순히 프롬프트로 "너는 기획자야" → "너는 레벨디자이너야"라고 전환하는 것은 실질적으로 사용자가 직접 명령하는 것과 차이가 없었음

#### 전환: Claude Code 서브에이전트(Subagent) 병렬 실행

이러한 한계를 해결하기 위해 **Claude Code의 서브에이전트 시스템**으로 전환했다. 서브에이전트는 각각 독립된 컨텍스트 윈도우를 가진 별도의 Claude 인스턴스로, 메인 에이전트(기획자)가 작업을 분해하여 각 서브에이전트에 위임하면 병렬로 실행된다. 각 에이전트는 자기 역할만 수행한 뒤 결과만 반환하므로, 메인 세션의 컨텍스트가 깨끗하게 유지된다.

커스텀 에이전트 등록은 CLI 실행 시 `--agents` 옵션을 활용했다. 이 방식은 에이전트 정의를 JSON 형태로 전달하여 현재 세션에서 바로 사용할 수 있으며, 에이전트 구성을 빠르게 테스트하고 조정하기에 적합하다. 게임 개발 특성상 작업 단계마다 에이전트 역할과 프롬프트를 유동적으로 수정해야 할 일이 많았기 때문에, `.claude/agents/`에 파일로 고정하기보다는 `--agents` 옵션으로 유연하게 구성을 바꿔가며 사용했다.

**에이전트 역할 구성:**

| 에이전트 | 구현 방식 | 역할 |
|---------|----------|------|
| AI 기획자 (메인) | Claude Code 메인 세션 | 핵심 피처 분석, 게임 설계 방향 수립. 현재 상황을 판단하여 서브에이전트에게 필요한 작업을 자율적으로 위임한다 |
| AI 아트 담당 | 서브에이전트 | 기획자의 지시를 받아 아트 프롬프트 생성, 나노바나나2로 에셋 출력 |
| AI 레벨디자이너 | 서브에이전트 | 기획자의 지시를 받아 UnityMCP를 통해 Unity 씬에 직접 맵 설계 |
| AI 개발자 | 서브에이전트 | 기획자의 지시를 받아 전투 로직, UI 연동, 사운드 연동 등 코드 구현 |

AI 기획자는 메인 세션에서 오케스트레이터 역할을 수행하며, 프로젝트 진행 상황에 따라 어떤 서브에이전트에게 어떤 작업을 위임할지 스스로 판단한다. 서브에이전트들은 독립된 컨텍스트에서 병렬로 작업을 수행하고, 완료된 결과만 메인 세션으로 반환한다.

**파이프라인 흐름:**

```
[AI 기획자 - 메인 세션] 핵심 피처 분석 및 요구사항 수립
    │ 상황에 따라 서브에이전트에 병렬 위임
    ├──────────────────┼──────────────────┤
    ↓                  ↓                  ↓
[서브에이전트]       [서브에이전트]      [서브에이전트]
 AI 레벨디자이너      AI 아트 담당        AI 개발자
 (독립 컨텍스트)      (독립 컨텍스트)     (독립 컨텍스트)
 UnityMCP로           아트 프롬프트       전투 로직 보완,
 맵 설계·             생성 및             UI 구현,
 프리팹 배치          나노바나나2         사운드 연동
                      에셋 출력
    ↓                  ↓                  ↓
    └──── 결과 반환 → 메인 세션에서 통합 ──┘
                       ↓
               [Suno] BGM/SFX 제작
```

#### 단일 세션 vs Claude Code 서브에이전트 비교

| 항목 | 단일 세션 (기존) | Claude Code 서브에이전트 (현재) |
|------|----------------|-------------------------------|
| 컨텍스트 | 모든 역할이 하나의 컨텍스트 공유 → 오염 | 각 에이전트가 독립 컨텍스트 보유 |
| 실행 방식 | 순차적 역할 전환 | 병렬 실행 가능 |
| 역할 분리 | 프롬프트로만 전환, 경계 모호 | `.claude/agents/`에 시스템 프롬프트 분리 정의 |
| 품질 | 대화 길어지면 저하 | 각 에이전트가 자기 역할에만 집중하여 품질 유지 |
| 작업 속도 | 에셋·레벨·코드를 순서대로 처리 | 동시 진행으로 작업 시간 단축 |

**2주차 작업 실적:**
- 단일 세션에서 프롬프트 기반 역할 전환으로 오케스트레이션 시도
- 컨텍스트 오염, 병렬 처리 불가 등 한계 체감
- Claude Code 서브에이전트로 전환하여 독립 컨텍스트 병렬 실행 구조 확립
- AI 레벨디자이너가 UnityMCP로 Unity 씬에 맵을 구성
- AI 아트 담당이 기획 의도에 맞는 프롬프트를 생성하고 에셋 출력을 수행

## 5. AI 활용 핵심 포인트 정리

이번 프로젝트에서 AI는 단순 보조 도구가 아니라 **제작 파이프라인의 핵심 구성요소**로 기능한다.

1. **단일 세션 → 서브에이전트 전환**: 초기에는 단일 세션에서 프롬프트 기반 역할 전환을 시도했으나, 컨텍스트 오염과 병렬 처리 불가 문제를 겪고 Claude Code 서브에이전트로 전환했다. 이를 통해 각 에이전트가 독립 컨텍스트에서 병렬로 작업하는 진정한 오케스트레이션을 실현했다.
2. **UnityMCP 직접 제작**: AI 레벨디자이너가 Unity 씬을 직접 조작하여 레벨을 설계함으로써, 기획-구현 사이의 간극을 최소화했다.
3. **나노바나나2 아트 파이프라인**: AI 아트 담당이 기획 요구사항을 바탕으로 프롬프트를 생성하고, 나노바나나2로 캐릭터·배경·UI·이펙트 에셋을 빠르게 생산한다.
4. **AI 개발자 코드 구현**: AI 개발자가 UI 연동, 사운드 연동, 로직 보완 등 코드 레벨의 작업을 수행하여 에셋과 시스템 간의 통합을 담당한다.
5. **Suno 사운드 파이프라인**: 게임 장면별 분위기를 텍스트 프롬프트로 정의하고, Suno로 BGM과 SFX를 생성하여 1인 개발에서 가장 접근하기 어려운 사운드 영역을 해결한다.
6. **AI 풀스택 제작**: 기획(Claude) → 레벨(UnityMCP) → 아트(나노바나나2) → 개발(Claude) → 사운드(Suno)로 이어지는 AI 풀스택 게임 제작 파이프라인을 실현한다.
