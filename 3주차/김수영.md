## 기술 스택
React Native + Expo + TypeScript + SQLite(expo-sqlite) 로 가세요.
이유:
- Expo는 iOS/Android 빌드 환경 세팅 지옥을 거의 다 없애줘요. Mac 없어도 iOS 빌드 가능(EAS Build).
- Claude Code가 가장 능숙하게 다루는 모바일 스택이에요. 학습 데이터가 압도적으로 많음.
- 친구한테 공유할 때 Expo Go 앱으로 QR코드만 찍으면 바로 실행돼요. 앱스토어 심사 없이 베타 테스트 가능.
- SQLite는 의존성 그래프 같은 관계형 데이터에 잘 맞고, 오프라인 우선이라 결혼 준비 같은 개인 데이터에 적합.

## 단계별 실행 계획
0단계: 사전 준비 (코딩 0줄)
먼저 이것부터 해두세요:

- Node.js LTS 설치 (이미 있으면 패스)

- GitHub 계정 — Claude Code랑 git 쓸 거니까

## 3단계: SPEC.md / CLAUDE.md 작성
Claude Code 안에서 첫 명령:

"이 프로젝트는 React Native + Expo + TypeScript로 만드는 결혼준비 일정 자동화 앱이야. 1단계로 SPEC.md와 CLAUDE.md를 작성하자. 아래 요구사항 반영해줘:
핵심 컨셉: 본식 날짜를 anchor로, 의존성 그래프 기반 역산 스케줄링. 태스크는 다른 태스크에 의존할 수 있음 (예: 네일은 웨딩촬영 기준 -2일).
데이터 모델: Rulebook(공통 템플릿, JSON으로 시드), UserPlan(사용자별 인스턴스), Task(개별 일정), Dependency.
저장소: expo-sqlite로 로컬 우선, 향후 클라우드 동기화 가능성 열어둘 것.
화면: 온보딩(본식 날짜 입력) → 캘린더 뷰 → 태스크 상세 → 룰북 편집 → 설정.
공개 배포 예정이라 다국어/계정/익명모드 구조는 나중에 붙일 수 있게 모듈 분리.
우선 SPEC.md 초안만 작성하고, 내가 검토한 후 코딩 시작할게."

CLAUDE.md에는 코드 스타일, 폴더 구조, 커밋 규칙을 적어두면 매 세션 일관성 유지됨.

## 5단계: MVP 화면 (이 순서대로)
각 단계마다 동작 확인하고 git commit:

온보딩 1화면: 본식 날짜 입력 → SQLite 저장
리스트 뷰: 계산된 태스크들이 날짜순으로 나열
캘린더 뷰: react-native-calendars로 월별 표시, 점으로 태스크 표시
태스크 상세: 탭하면 상세 모달, 완료 체크, 메모
룰북 편집: 사용자가 자기 상황에 맞게 추가/삭제/수정

여기까지가 v0.5. 이 시점이 친구한테 처음 공유할 타이밍이에요.
